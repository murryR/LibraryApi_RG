# LibraryApi Documentation

Documentation for the LibraryApi solution is organized as follows:

| Document | Description |
|----------|-------------|
| [QUICK_START.md](QUICK_START.md) | Short overview, stack, and main features |
| [ARCHITECTURE.md](ARCHITECTURE.md) | Architecture, layers, and developer guide |
| [API_REFERENCE.md](API_REFERENCE.md) | REST API reference (API Key access) |
| [UI_GUIDE.md](UI_GUIDE.md) | End-user guide for the Blazor web UI |
| [DEPLOYMENT_AZURE.md](DEPLOYMENT_AZURE.md) | Deploying to Azure App Service |

## For developers

- Start with **QUICK_START.md** for a high-level picture.
- Use **ARCHITECTURE.md** for layer design, configuration, logging, and testing.
- Use **API_REFERENCE.md** when integrating with the API (API Key).

## For operators

- Use **DEPLOYMENT_AZURE.md** for build, deploy, and troubleshooting on Azure.
